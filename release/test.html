📋 Important: Data Structure Differences

🔄 API Requests (What YOU send):

{
  "customer_name": "John Doe",
  "table_number": 5,
  "items": [...],
  "total": 22.97
}
Note: Send raw order data directly, NOT wrapped in "payload"

📡 WebSocket Events (What you RECEIVE):

{
  "order_id": "ORD-ABC123",
  "payload": {
    "customer_name": "John Doe",
    "table_number": 5,
    "items": [...],
    "total": 22.97
  }
}
Note: WebSocket wraps your data in "payload" + adds "order_id"

➕ Create Order

📍 Endpoint: POST /api/v1/orders

📤 Send: Raw order data (just your order object)

📥 Get: { "success": true, "order_id": "ORD-XXXXX" }

🔔 WebSocket Broadcast: { "order_id": "ORD-XXX", "payload": { your_order_data } }

✏️ Update Order

📍 Endpoint: PUT /api/v1/orders/{order_id}

📤 Send: Raw updated order data (just your order object)

📥 Get: { "success": true, "order_id": "ORD-XXXXX" }

🔔 WebSocket Broadcast: { "order_id": "ORD-XXX", "payload": { your_updated_data } }

🗑️ Delete Order

📍 Endpoint: DELETE /api/v1/orders/{order_id}

📤 Send: Order ID in URL (no body needed)

📥 Get: { "success": true, "order_id": "ORD-XXXXX" }

🔔 WebSocket Broadcast: { "order_id": "ORD-XXX", "payload": { deleted_order_data } }
